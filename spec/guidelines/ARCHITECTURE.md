# goapk ‚Äî Architecture

## Core Principle

`goapk` is a single-binary CLI. All logic lives in `internal/`. `main.go` dispatches subcommands ‚Äî nothing else.

## Package Layout

```
main.go                  Entry point, subcommand dispatch (build, keygen)
internal/
  cli/                   Flag parsing, options structs
  config/                BuildConfig: validated build parameters
  manifest/              Web App Manifest (manifest.json) parser
  icon/                  Icon resizing: color + monochrome, all density buckets
  xmlbin/                Android binary XML encoder (AndroidManifest.xml)
  res/                   resources.arsc generator
  embed/                 go:embed declarations for pre-compiled artifacts
  zip/                   APK ZIP assembly (entry ordering, compression rules)
  align/                 zipalign: 4-byte alignment of uncompressed entries
  sign/                  APK Signature Scheme v2 + RSA keystore generation
  build/                 Pipeline orchestrator: wires all stages together
java/
  WebViewActivity.java   Source for the pre-compiled WebView activity DEX
tools/
  genstub/               Generates the minimal DEX stub (no classes)
testdata/
  pwas/                  Sample minimal PWAs for integration tests
```

## APK Assembly Pipeline (internal/build)

1. Icon processing (`internal/icon`) ‚Äî resize source PNG to 5 densities (mdpi‚Üíxxxhdpi)
2. Binary XML encoding (`internal/xmlbin`) ‚Äî encode AndroidManifest.xml
3. Resource table generation (`internal/res`) ‚Äî build resources.arsc
4. DEX embedding (`internal/embed`) ‚Äî copy pre-compiled classes.dex
5. Web asset packing ‚Äî copy assets into assets/www/; write assets/config.json for URL mode
6. ZIP assembly (`internal/zip`) ‚Äî build APK with correct entry order and compression
7. zipalign (`internal/align`) ‚Äî align uncompressed entries to 4-byte boundaries
8. APK signing (`internal/sign`) ‚Äî sign with APK Signature Scheme v2

## Dependency Rules

- `build` is the only package that imports across domains
- `main.go` imports `build`, `config`, `cli`, `sign`
- `xmlbin`, `res`, `icon`, `zip`, `align`, `sign`, `manifest`, `embed` must not import each other
- `config` may import `manifest`

## Manifest Metadata Priority

1. CLI flags (highest priority ‚Äî always override)
2. `manifest.json` fields (if `--manifest` provided or auto-detected in assets dir)
3. Defaults

## Resource ID Conventions

App resources use package 0x7f:
- 0x7f010000 = @string/app_name
- 0x7f020000 = @mipmap/ic_launcher (color, per-density via config)
- 0x7f020001 = @mipmap/ic_launcher_mono (monochrome, per-density via config)

Android framework attribute IDs (from AOSP DumpManifest.cpp):
- android:label = 0x01010001
- android:icon = 0x01010002
- android:name = 0x01010003
- android:exported = 0x01010010
- android:versionCode = 0x0101021b
- android:versionName = 0x0101021c
- android:minSdkVersion = 0x0101020c
- android:targetSdkVersion = 0x01010270
- android:allowBackup = 0x01010280

## DEX Artifacts

`internal/embed/classes.dex` is committed to the repo. Two versions:
1. **Stub** (default): valid DEX header + map, no classes. Generated by `tools/genstub`.
   APK assembles but crashes on device.
2. **Real** (after `make dex`): compiled from `java/WebViewActivity.java` using Android SDK `d8`.
   Required for a functional APK.

The `goapk` binary embeds whichever version is in `internal/embed/classes.dex` at build time.
